# 재활용 순환 플랫폼

## 프로젝트 개요

재활용 비율을 높여 매립 악화 문제를 줄이고, 재활용품을 재질로 바꾸어 판매하는 공장과 재활용 재질로 사업을 하는 기업에게 정보를 제공하며 산업 홍보를 지원하는 플랫폼입니다.

일반인이 버린 쓰레기의 대량 매립 문제를 완화하고, 물질 자원 부족을 예방하여 사회적 도움을 주는 것을 목적으로 합니다. 일회성이 아닌 N회성으로 만들어 지속 가능한 재활용 순환 생태계를 구축합니다.

## 만든 목적과 기여법

기업과 일반인의 통계 지수를 이용해 정보와 방법, 지식을 오픈하여 활발한 재활용과 순환을 이끌고, 기업이 재활용품을 활용하여 절약하는 방법을 제시합니다.

**핵심 목표:**
- 다 함께 움직이는 쓰레기 재질 순환 생태계 구축  
- 재활용 비율 향상을 통한 매립 악화 문제 완화  
- 재활용품을 재질로 바꾸어 판매하는 공장 정보 지원  
- 재활용 재질로 사업을 하는 기업에 산업 홍보 지원
- 재활용 공장과 사업자 연결을 위한 정보 지원  
- 일반인이 버린 쓰레기 대량 매립 문제 완화  
- 물질 자원 부족 예방  

## 주요 기능

### 1. 홈 화면에 정보 채우기

재활용 → 재질별 → 쓰레기 순환 과정을 시각적으로 제공합니다.

- **재활용 과정**: 재활용품 수거부터 처리까지의 전체 과정
- **재질별 분류**: 플라스틱, 종이, 유리, 금속 등 재질별 재활용 정보
- **쓰레기 순환**: 어떤 공장에서 어떻게 output 되고, 어떻게 기업으로 가는지 설명
- **순환된 제품 예시**: 재활용된 재질로 만들어진 제품 사례를 메인 화면에서 제공

### 2. 활동 포인트 = 프로필 아이콘

- **소셜 로그인 데이터**: 소셜 로그인을 통한 사용자 인증 및 계정 정보 관리
- **일일 포인트 시스템**: 매일 12시마다 버튼을 누르면 1포인트 획득
- **재활용 물품 프로필**: 소셜 로그인 데이터의 계정 정보를 이용해 재활용된 물품(예: PET병)에서 재활용 물품 회원 프로필을 가질 수 있음

### 3. 카테고리, 아이디어 통계

- **정책/기술 아이디어 게시판**: 정책 제안 및 기술 아이디어를 작성할 수 있는 게시판
- **좋아요/싫어요 기능**: 각 아이디어에 대한 사용자 의견 수집
- **통계 분석**: 카테고리별 관심도 통계 제공

### 4. 게시판

- **커뮤니티 게시판**: 사용자 간 정보 공유 및 소통 공간
- **카테고리 관심 통계**: 게시판 내 카테고리별 관심도 및 통계 정보 제공

## 기술 스택

### Backend
- **Java 21**: 최신 Java 버전 사용
- **Spring Boot 3.5.5**: 웹 애플리케이션 프레임워크
- **Spring Data JPA**: 데이터베이스 ORM
- **H2 Database**: 인메모리 데이터베이스 (개발용)
- **Lombok**: 보일러플레이트 코드 감소

### Frontend
- **Mustache**: 서버 사이드 템플릿 엔진
- **Bootstrap 5**: 반응형 UI 프레임워크

### Build Tool
- **Gradle**: 빌드 및 의존성 관리

## 실행 방법

1. **프로젝트 클론**
   ```bash
   git clone [repository-url]
   cd A_JavaPy_PreOrder/SpringWeb
   ```

2. **Gradle 빌드**
   ```bash
   ./gradlew build
   ```

3. **애플리케이션 실행**
   ```bash
   ./gradlew bootRun
   ```

4. **웹 브라우저에서 접속**
   ```
   http://localhost:8080
   ```

## 주요 엔드포인트

- `/` - 홈 화면 (소개 페이지)
- `/community` - 커뮤니티 게시판
- `/emp` - 게시글 등록 폼
- `/idea` - 아이디어 투표 페이지
- `/data` - API 환경데이터 시각화
- `/idea/like/{id}` - 아이디어 좋아요
- `/idea/dislike/{id}` - 아이디어 싫어요

## 데이터베이스

H2 인메모리 데이터베이스를 사용하며, 애플리케이션 실행 시 자동으로 테이블이 생성됩니다.

H2 콘솔 접속: `http://localhost:8080/h2-console`
